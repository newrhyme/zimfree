# ZimFree (짐프리) — 부산 지하철 여행 코파일럿

> "내 짐과 배터리 상태까지 아는 부산 지하철 여행 어시스턴트"

DIVE 2026 해커톤 (부산교통공사 × 짐캐리 트랙) 출품작.
캐리어·유모차 등 짐을 든 관광객에게 **탑승칸 단위의 경로 안내, 혼잡 시간 회피,
엘리베이터 고장 시 대체경로, 짐 처리(휴대/보관함/짐배송) 의사결정**을 제공하는 웹 서비스 MVP.

## 심사 기준 (모든 개발 판단의 우선순위)

| 항목 | 배점 | 개발 시사점 |
|---|---|---|
| 데이터 기반 문제해결 | **30** | 제공 데이터 13종을 실제로 조인·모델링에 사용해야 함. 데이터 발명 금지 |
| 과제 이해도 | 20 | 3대 pain point(대형 수하물/역사 내 이동·환승/편의시설 정보 부족)에 기능이 1:1 대응 |
| 창의성·AI 혁신성 | 20 | ML 모델 2종 + LLM 에이전트. "LLM에 CSV 통째로" 금지 |
| 서비스 완성도 | 20 | 데모 시나리오 5종이 끊김 없이 시연되는 것이 목표 |

## 모노레포 구조

```
zimfree/
├── CLAUDE.md            ← 이 파일. 데이터 계약과 전역 규칙
├── README.md            ← 사람/심사위원용 소개
├── .claude/settings.json
├── Makefile             ← setup / data / train / dev
├── data/
│   ├── raw/             ← 원본 CSV 13종 (읽기 전용, 절대 수정·삭제 금지)
│   └── processed/       ← 전처리 산출물 (app.db, *.parquet) — 스크립트로만 생성
├── backend/             ← FastAPI + ML. backend/CLAUDE.md 참조
└── frontend/            ← Vite + React. frontend/CLAUDE.md 참조
```

## 확정 기술 스택 (임의 변경 금지)

- **Backend**: Python 3.11+, FastAPI, uvicorn, pandas, **LightGBM**, scikit-learn, SQLite, `anthropic` SDK
- **Frontend**: Vite + React 18 + TypeScript, Tailwind CSS, react-leaflet(OSM 타일, API 키 불필요)
- **LLM**: Anthropic Messages API, 모델 `claude-sonnet-4-6` (환경변수 `ANTHROPIC_API_KEY`, 없으면 어시스턴트 기능만 graceful degrade — 나머지 기능은 키 없이 전부 동작해야 함)
- 패키지 관리: backend는 `uv`(없으면 pip+venv), frontend는 `npm`

## 데이터 계약 — data/raw/ 13종

모든 파일 **UTF-8 (BOM)**, `encoding="utf-8-sig"`로 읽는다.
**조인 키는 `역번호`(int)를 최우선**으로, 역번호가 없는 파일(설비 4종)은 `표준역명`을 사용한다.
`역명`은 표기 변형이 있으므로 조인에 쓰지 말고 표시용으로만 쓴다.
환승역(서면·연산·동래·덕천 등)은 호선별로 역번호가 다르다(예: 서면 = 119, 219). "역 전체" 수치가 필요하면 표준역명으로 groupby 후 합산한다.

**권역→대표역 매핑** (flow 파일 2행째에 명시됨, 상수로 코드화):
해운대·기장→해운대역 / 광안리→광안역 / 서면·부산진구→서면역 / 원도심(동구·중구)→중앙역 / 기타→매핑 없음.
목적지 역이 속한 권역 판정은 이 대표역과의 유클리드 거리(위경도)로 최근접 권역을 취한다.

## 전역 규칙

1. **데이터 발명 금지.** raw에 없는 수치·역·시설을 만들지 않는다. 커버리지 밖이면 UI에 "정보 없음"을 정직하게 노출한다.
2. `data/raw/`는 읽기 전용. 전처리 결과는 반드시 `data/processed/`에 스크립트로 생성 (재현 가능해야 함).
3. 컬럼명은 원본 한글 그대로 읽고, processed 테이블/API부터는 영문 snake_case로 변환한다 (매핑은 `backend/app/schema.py` 한 곳에서만 정의).
4. 시간대 컬럼 `24시-01시`는 익일 01시로 취급하지 말고 당일 hour=24 슬롯으로 단순 처리한다 (MVP 범위).
5. UI 문구는 한국어. 코드 식별자·주석은 영어.
6. 커밋은 기능 단위로 작게, conventional commits(`feat:`, `fix:`, `data:`, `docs:`).
7. 각 단계 완료 시 테스트(`make test`)가 통과해야 다음 단계로 진행.

## 빌드 순서 (Claude Code 작업 계획)

1. `Makefile` + backend 스캐폴드 → `backend/CLAUDE.md`의 전처리 파이프라인 구현 → `data/processed/app.db` 생성
2. 모델 2종 학습 스크립트 (`train_congestion.py`, `train_route_difficulty.py`) → `backend/models/` 아티팩트
3. FastAPI 엔드포인트 (backend/CLAUDE.md 스펙 준수) + pytest
4. Frontend 4개 화면 (frontend/CLAUDE.md) + Vite proxy로 백엔드 연결
5. 데모 시나리오 5종 E2E 점검

## MVP Definition of Done — 데모 시나리오 5종

1. **탑승칸 추천**: 부산역→해운대, 28인치 캐리어 2개 입력 → "6-1 위치 탑승" 카드 + 도착역 엘리베이터 위치 + 연단간격/곡선 경고가 뜬다.
2. **혼잡 회피**: 서면 환승 포함 경로에서 요청 시각의 혼잡 레벨과 "덜 붐비는 출발 시간" 대안이 표시된다.
3. **고장 대응**: 고장 시뮬레이션 토글(실데이터 미운행 역 중 하나 선택) → 대체경로 모델이 재계산한 경로와 복잡도 등급이 표시된다. ALT_NONE 구간은 "이동 불가" 경고.
4. **배터리 모드**: 배터리 15% 입력 → 경로 상 충전설비 위치·요금 안내가 포함된다.
5. **어시스턴트**: "해운대 가는데 캐리어 두 개야, 짐 어떻게 할까?" 자연어 질의에 tool-use로 위 기능들을 호출한 종합 답변이 온다. (API 키 없으면 이 시나리오만 스킵 가능)
