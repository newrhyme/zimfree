# 짐프리 ZimFree — 부산 지하철 여행 코파일럿

> 내 짐과 배터리 상태까지 아는 부산 지하철 여행 어시스턴트
> DIVE 2026 · 부산교통공사 × 짐캐리 트랙

기존 지도 앱은 "가장 빠른 길"을 알려주지만, 캐리어를 끄는 관광객에게 필요한 건
**"짐을 들고 갈 수 있는 길"** 입니다. 짐프리는 부산교통공사 공공데이터 10종과
짐캐리 민간데이터 3종을 융합해 짐 든 여행자의 이동 전 과정을 안내합니다.

## 핵심 기능

| 기능 | 설명 | 사용 데이터 |
|---|---|---|
| 🚪 탑승칸 추천 | 내릴 역 엘리베이터 바로 앞 칸(예: 6-1)을 추천, 연단간격·곡선구간 경고 | 엘리베이터 상세위치, 승강장 간격·곡선 4,864건 |
| 📊 혼잡 회피 | 역×요일×시간대 혼잡 예측(LightGBM), 덜 붐비는 출발 시간 제안 | 2025년 승하차 81,760행 |
| 🛗 고장 대응 | 엘리베이터 고장 시 대체경로와 난이도 등급을 ML로 재계산 | 대체경로 932건(학습라벨 포함) |
| 🔋 배터리 모드 | 저배터리 시 경로 상 휴대폰 충전설비 안내 | 충전설비 108개 역 |
| 🧳 짐 처리 결정 | 휴대 vs 보관함 vs 짐캐리 배송을 데이터 근거와 함께 비교 추천 | 보관함 현황 + 짐배송 OD 흐름 |
| 💬 AI 어시스턴트 | 위 기능을 도구로 쓰는 자연어 대화 (Claude tool-use) | 전체 |

## 아키텍처

```
React(Vite) ── /api proxy ──> FastAPI ──> SQLite(app.db)
                                 ├─ LightGBM 혼잡도 회귀
                                 ├─ LightGBM 대체경로 난이도 분류
                                 └─ Claude Messages API (tool-use)
        원본 CSV 13종 ─ preprocess.py ─> app.db (재현 가능 파이프라인)
```

## 빠른 시작

```bash
# 0) 데이터 배치: 제공받은 *_revised.csv 13종을 data/raw/ 에 복사

# 1) 전체 셋업 + 전처리 + 모델 학습
make setup && make data && make train

# 2) 개발 서버 (backend :8000, frontend :5173)
make dev

# (선택) AI 어시스턴트 활성화 — 둘 중 하나만 있으면 됨 (OpenAI 우선)
export OPENAI_API_KEY=sk-...          # GPT (기본 모델 gpt-4o-mini, OPENAI_MODEL로 변경)
# 또는
export ANTHROPIC_API_KEY=sk-ant-...   # Claude (claude-sonnet-4-6)

# 3) 테스트 (backend pytest + frontend 빌드)
make test
```

## 데모 시나리오 (E2E 검증됨)

| # | 시나리오 | 확인 포인트 |
|---|---|---|
| 1 | 부산역→해운대, XL 캐리어 2개 | 탑승칸(5-2) + 도착역 엘리베이터 위치 + 경고 카드 |
| 2 | 서면 금요일 18시 | 혼잡 레벨(혼잡↑) + 덜 붐비는 시간 대안 |
| 3 | 다대포해수욕장 EV 고장 | ALT_NONE 구간 "이동 불가" (실데이터) |
| 4 | 배터리 15% | 경로 상 충전설비·요금 안내 |
| 5 | 자연어 질의 | tool-use 종합 답변 (`ANTHROPIC_API_KEY` 필요, 없으면 스킵) |

## 데이터 출처·정책

- 부산교통공사 제공 공공데이터 10종, 짐캐리 제공 민간데이터 3종 (DIVE 2026 개방 데이터셋)
- 원본은 `data/raw/`에 읽기 전용 보관, 모든 가공은 `scripts/preprocess.py`로 재현 가능
- 제공 데이터에 없는 수치는 서비스에 표시하지 않습니다 (커버리지 밖 = "정보 없음")

## 개발 방식

이 저장소는 Claude Code가 `CLAUDE.md`(루트·backend·frontend)의 스펙을 따라
자율 개발하도록 설계되었습니다. 사람의 역할은 스펙 갱신과 데모 검수입니다.
